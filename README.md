### Цель работы:
Исследование и разработка подхода к автоматической генерации unit-тестов для С++ на основе анализа исходного кода.

### Задачи:
1. Изучить основные подходы к тестированию программного обеспечения
2. Проанализировать существующие инструменты автоматической генерации тестов
3. Изучить методы анализа исходного кода
4. Разработать модуль парсинга C++ кода с целью извлечения информации о функциях.
5. Реализовать алгоритм генерации тестовых случаев на основе полученных данных о функциях.
6. Разработать механизм формирования тестов в формате фреймворка Google Test.
7. Провести тестирование разработанного решения.
8. Проанализировать полученные результаты и оценить эффективность разработанного решения.

### Обзор предметной области

Сущетсвует довольно много программ для автоматического написания тестов на C++. Их можно разделить на несколько категорий:
1. Промышленная разработка (Parasoft, Cantata). Такие инструменты сложны в освоении и дороги в использовании для небольших и средних проектах
2. Coyote C++ - исследовательский проект с высоким порогом входа
3. AI-инструменты (Copilot, Qodo). Пользуются большой популярностью


### Ожидаемый результат
В рамках данной работы планируется разработка инструмента, осуществляющего автоматическую генерацию unit-тестов на основе анализа исходного кода на языке C++.

Разрабатываемый инструмент должен быть простым в установке и использовании, что позволит применять его как в учебных, так и в практических задачах.

##### Основной функционал инструмента включает:
1. Выполнение анализа исходного кода на основе абстрактного синтаксического дерева (AST)
2. Выделение ключевых конструкций (функции, условия, ветвления)
3. Автоматическая генерация unit-тесты для отдельных функций и методов
4. Формирование тестов в формате, совместимом с Google Test
5. Оценку покрытия кода сгенерированными тестами

Важно отметить, что для создания максимально корректных и релевантных тестов участие разработчика остаётся необходимым. Разрабатываемый инструмент направлен на упрощение и ускорение процесса тестирования, однако не заменяет полностью ручную разработку тестов.
